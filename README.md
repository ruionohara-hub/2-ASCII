# 2-ASCII

制作期間 : 約1ヵ月

アスキーアート変換ツールは、任意の画像ファイル(PNG, JPGなど)を読み込み、指定した文字セットを用いてカラーのアスキーアート(AA)テキストおよび画像データとして変換・保存できるデスクトップアプリケーションです。ターミナルライクな表現やドット絵風のアート制作など、デジタル画像に新たな視覚的アプローチを提供します。

## 技術的な工夫点

本アプリケーションは、単なるテキスト変換ツールではなく多様な環境での安定動作と、画像としての出力品質にこだわって開発しました。

* **輝度ベースの文字マッピングとカラー情報の独立保持**
    画像を単に白黒にするだけでなく、ピクセルの輝度(明るさ)に基づいてASCII文字を割り当てる一方で、元のRGBカラー情報を独立したマトリックスとして保持する設計にしました。これにより、後段のレンダリング処理で鮮やかなフルカラーAA画像を生成可能にしています。
  
* **OSに依存しない等幅フォントの動的探索**
    AAの崩れを防ぐためには等幅フォントが必須でした。OS環境(Windows/macOS/Linux)に応じた最適な等幅フォント(Consolas, Menlo, DejaVuSansMonoなど)を自動で探索し、見つからない場合でも代替フォントやPillowのデフォルトフォントを使用する強固なフォールバック処理を実装しました。
  
* **文字のバウンディングボックスを用いたアスペクト比の動的補正**
    テキスト文字は縦長の長方形であるため、単純に出力すると画像全体が縦に間延びしてしまいます。これを防ぐため、フォントのbboxから実際のピクセルサイズを動的に計算し、元画像のアスペクト比を正確に維持する高度な補正処理を実装しました。

## ユーザー観点での工夫

「表現したいテイストに合わせて直感的に試行錯誤できる」ことを重視し、UI/UXを設計しました。

* **リアルタイムなテキストプレビュー機能**
    変換結果を画像として書き出す前に、GUI上のテキストボックスで即座にプレビューできるようにしました。結果に確認後に保存できるため、作業の手間を大幅に省きます。
  
* **カスタマイズパラメータ**
    「変換幅(文字数)」や「使用する文字セット」をユーザーが自由に指定できるUIにしました。デフォルトの文字セットだけでなく、独自の記号を入力することで、表現したいディテールに合わせてユーザー自身が最適な出力を調整できるようにしています。
  
* **用途に合わせた比率オプション**
    AA特有の縦長感をあえて活かす「AAデフォルト補正」と、元の写真の比率を忠実に再現する「元のアスペクト比」の2つのモードを用意し、使用者の目的に応じた出力を簡単に選択可能にしました。

## 課題に対するアプローチ

* **巨大画像生成時のクラッシュ防止**
    変換幅（文字数）を極端に大きく設定した場合、書き出す出力画像サイズが膨大になりプログラムがメモリ不足でクラッシュするリスクがありました。そのため、内部で出力サイズに上限を設けるとともに、画像サイズが無効・小さすぎる場合には自動で最小サイズに調整するセーフティネットを実装し、堅牢性を高めました。
  
* **ライブラリのバージョン差異への対応**
    画像描画ライブラリ(Pillow)のバージョンによって、フォントサイズの取得メソッドが非推奨・廃止になるなどの互換性の問題がありました。これに対し、複数の取得メソッドをtry-except構文で順番に試行する多段フォールバック処理を実装し、ユーザーのPC環境に依存せず安定して動作するよう解決しました。

## エピソード

* **制作の流れ、きっかけ、得たこと**
    ネット文化の象徴であるアスキーアート(AA)を誰でも簡単に作れたら面白いと思い、制作しました。画像から出力できるようにするため、まずは様々な画像を用意し、画像による出力後の結果の違いなどをまとめて、その後改善していく流れで制作を行っていました。初期の段階ではプレビューを搭載しておらず、結果は出力された画像で確認していました。すべての変換作業をこのexeで完結させるためにオプションを多く搭載し、他と区別させるため、文字色の追加も行いました。その段階で制作の作業効率が上がったため、今では検証がスムーズに行えるような作り方を意識しています。
